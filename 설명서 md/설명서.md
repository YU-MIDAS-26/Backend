## 프로젝트 개요

**YU-MIDAS-Backend** — 농산물/식재료 시세 정보 수집 및 제공 백엔드 서버

- 매일 KAMIS(농산물유통정보) API에서 시세 자동 수집 → MySQL 저장
- 프론트엔드는 저장된 DB 데이터를 API로 조회
- 네이버 쇼핑 API로 최저가 검색 기능 포함

## DB 테이블

### market_prices (농산물 시세)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | BIGINT PK | 자동증가 |
| item_name | VARCHAR(100) | 품목명 (예: 배추, 양파) |
| avg_price | DECIMAL(15,2) | 평균가격 |
| min_price | DECIMAL(15,2) | 최저가격 |
| max_price | DECIMAL(15,2) | 최고가격 |
| unit | VARCHAR(30) | 단위 (예: 10kg) |
| market_name | VARCHAR(100) | 품종/등급 (예: 상품) |
| collected_date | DATE | 수집 날짜 |
| source | VARCHAR(100) | 출처 (`kamis` 고정) |
| category_code | VARCHAR(10) | 부류코드 |
| created_at | DATETIME | 생성일시 |
| updated_at | DATETIME | 수정일시 |

**categoryCode 값:**

| 코드 | 분류 |
|------|------|
| 100 | 식량작물 (쌀, 보리 등) |
| 200 | 채소류 (배추, 양파, 무 등) |
| 300 | 특용작물 (참깨, 들깨 등) |
| 400 | 과일류 (사과, 배, 감귤 등) |
| 500 | 축산물 (소고기, 돼지고기 등) |
| 600 | 수산물 (고등어, 명태 등) |

### ingredients (식재료)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | BIGINT PK | 자동증가 |

| unit | VARCHAR(30) | 단위 |

## API 명세

### 1. 시세 API `/api/prices`

#### 시세 수동 수집 (관리용)
```
POST /api/prices/collect?categoryCode=200
```
- `categoryCode`: 100~600 (기본값: 200 채소류)
- `productClsCode`: 01(소매), 02(도매) 선택
- `countryCode`: 지역코드 (기본: 1101 서울)

응답:
```json
{
  "success": true,
  "message": "시세 수집이 완료되었습니다.",
  "data": { "savedCount": 42 }
}
```

#### 카테고리/품목명으로 시세 목록 조회
```
GET /api/prices/list?categoryCode=200&itemName=배추
```
- `categoryCode`: 선택 (없으면 전체)
- `itemName`: 선택, 부분 검색 (없으면 전체)
- 최대 100건, 최신순 정렬

응답:
```json
{
  "success": true,
  "message": "요청이 성공적으로 처리되었습니다.",
  "data": [
    {
      "id": 1,
      "itemName": "배추",
      "avgPrice": 3500.00,
      "minPrice": 3200.00,
      "maxPrice": 3800.00,
      "unit": "10kg",
      "marketName": "상품",
      "collectedDate": "2026-04-08",
      "source": "kamis",
      "categoryCode": "200"
    }
  ]
}
```

#### 특정 품목 최신 시세
```
GET /api/prices/latest?itemName=배추
```

#### 특정 품목 최저가
```
GET /api/prices/best?itemName=배추
```

#### 최근 수집 데이터 100건
```
GET /api/prices/recent
```

## 스케줄러

- **파일**: `domain/price/service/MarketPriceService.java` (148번째 줄)
- **실행 시간**: 매일 09:11 (`application-local.properties`의 `kamis.collect.cron=0 11 9 * * *`)
- **동작**: 부류코드 100, 200, 300, 400, 500, 600 전체를 순서대로 수집하여 DB 저장
- cron 형식: `초 분 시 일 월 요일`

**주의사항:**
- KAMIS API는 당일 오전 중에는 데이터가 없을 수 있음 (오후에 업데이트)
- `p_regday` 파라미터에 오늘 날짜가 자동으로 세팅됨

## 앞으로 추가 가능한 기능

### 가격 추이 API (미구현)

**Repository** 추가:
```java
List<MarketPrice> findByItemNameAndCollectedDateBetween(
    String itemName, LocalDate start, LocalDate end);
```

**Service** 추가:
```java
public List<MarketPriceResponse> getTrend(String itemName, int days) {
    LocalDate end = LocalDate.now();
    LocalDate start = end.minusDays(days);
    return repository.findByItemNameAndCollectedDateBetween(itemName, start, end)
        .stream().map(MarketPriceResponse::from).toList();
}
```

**Controller** 추가:
```java
@GetMapping("/trend")
public ApiResponse<List<MarketPriceResponse>> trend(
    @RequestParam String itemName,
    @RequestParam(defaultValue = "30") int days) {
    return ApiResponse.success(service.getTrend(itemName, days));
}
```

호출: `GET /api/prices/trend?itemName=배추&days=30`

## 주요 개발 이슈 및 해결 기록

| 이슈 | 원인 | 해결 |
|------|------|------|
| KAMIS API 302 리다이렉트 | `http://` 사용 | `https://`로 변경 |
| KAMIS savedCount:0 | `p_regday` 파라미터 누락 | 오늘 날짜 추가 |
| KAMIS content-type 파싱 오류 | JSON이 아닌 content-type 반환 | `bodyToMono(String.class)` + 수동 파싱 |
| MySQL Access Denied | MariaDB가 포트 3306 선점 | MariaDB 종료 후 MySQL 실행 |
| MySQL TCP 인증 실패 | `--skip-grant-tables` 프로세스 충돌 | 해당 프로세스 kill 후 정상 실행 |
| MySQL 8.4 native_password 없음 | 8.4에서 해당 플러그인 제거됨 | `caching_sha2_password` 사용 + `allowPublicKeyRetrieval=true` |
